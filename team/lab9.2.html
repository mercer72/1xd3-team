    Purpose

    • Rentta is a user driven market for exchanging and borrowing items. Rentta is inspired by online stores like Amazon, but with an important difference; Rentta is only for free and or affordable items. “Trust” is what is most valuable to us, and you will be able to get “trust” currency from other users, the point is so you know if the person you are lending an item will return it or not. Rentta will have no online exchange of currency, and likewise no shipping services built in, interactions are purely user based, the primary role of Rentta is to be an advertisement for user goods.

        Data Type INT VARCHAR(15) VARCHAR(300) VARCHAR(300) INT

    • Listings Table
        • Field Name listing_id user_id contact_info title description type status keywords
        Data Type INT INT VARCHAR(320) VARCHAR(200) TEXT VARCHAR(20) VARCHAR(20) TEXT
        Use A unique Id for the listing The owners Id A method of contact The title of a item The description of the item Borrow, or Exchange The status of the item Keywords uses for searching

        Primary Key: listing_id

        Foreign Key: user_id

    Functions performed by the app

    • makeAccount
      • Database effect: INSERT, adds an account to the user table

    • deleteAccount
      • Database effect: DELETE, deletes an account from the user table

    • makeListing
      • Database effect: INSERT, adds a listing to the listings table

    • deleteListing
      • Database effect: DELETE, deletes a listing from the listings table

    • confirmListing
      • Database effect: UPDATE, adds trust to the user's involved updating the user table
      • Database effect: DELETE, deletes the listing from the listing table

    • login
      • Database effect: SELECT, selects the user from the user table and logs them in

    • showListing
      • Database effect: SELECT, selects the clicked on listing from the listings table and shows it to the client

    • search
      • Database effect: SELECT, will show listings based on keywords from the listings table

    • logout
      • Database effect: none, logs the user out

    Client Feedback

    • "Good job, some suggestions: To create a good shopping experience, ensure the navigation bar maintains a consistent height and width across all pages to prevent layout shifts, and standardize your paragraph styles to maintain a clean, rhythmic UI. A critical functional improvement for any e-commerce or registration site is the implementation of anchor links or interactive image triggers; when a user clicks a product image or a "View Details" button, the page should immediately jump to the specific section containing the price, product details, etc."
    • Based on the client's feedback, first, the suggestion to “ensure the navigation bar maintains a consistent height and width across all pages to prevent layout shifts” means we need to standardize our layout structure. Currently, if our navbar changes size between pages, it can create a jarring experience for users. To fix this, we will define a fixed height and consistent styling in our CSS (e.g., using a shared class or global stylesheet) so every page feels stable and predictable.
    • Next, the recommendation to “standardize your paragraph styles to maintain a clean, rhythmic UI” impacts our typography. Right now, inconsistent font sizes, spacing, or alignment can make the site look unpolished. We will address this by creating consistent rules for text (such as font-size, line-height, margins) to ensure readability and visual harmony across all sections.
    • Finally, the most important functional improvement is implementing interactivity: “when a user clicks a product image or a 'View Details' button, the page should immediately jump to the specific section containing the price, product details, etc.” This means we need to introduce anchor links or clickable elements that improve navigation within the page. Instead of forcing users to scroll manually, we will connect buttons or images to specific sections using IDs (e.g., href="#product-details"). This significantly enhances user experience by making the site feel more responsive and intuitive.
    • Overall, these suggestions push us to improve both consistency (design/UI) and usability (navigation/functionality), bringing our project closer to a professional e-commerce standard.
